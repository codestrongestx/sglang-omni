1. Golden-trace harness. Instrument the minisgl reference to dump intermediate artifacts at every stage seam for a fixed input set + seed: conditioning tokens, the speaker-embedding vector, the full 9×T codebook token grid + eos_frame, and final PCM. Evaluated by: re-running produces byte-identical artifacts — these become the oracle for every step below.
2. Topology skeleton. Declare the 4-stage DAG in Omni — stage types, the tensor-shape contract at each handoff, inbox/outbox connections — with all stages as passthrough stubs. Evaluated by: a dummy payload traverses all four stages end-to-end and the server responds. Plumbing is proven correct before any model logic exists.
3. Text-frontend stage. Real normalization → UTF-8 byte tokenization → conditioning-token assembly, with NeMo TN as a lazy/optional component. Evaluated by: emitted token sequence matches the golden trace exactly, token-for-token.
4. Speaker-embedding stage. ECAPA-TDNN + LDA projection, with per-speaker caching. Evaluated by: embedding cosine-similarity ≈ 1.0 against the golden vector, and a cache hit returns the identical vector.
5. LM decode — teacher-forced correctness. Stand up the MoE backbone + multi-codebook head, then feed the reference's exact input frames under teacher forcing (no sampling). Evaluated by: per-frame logits / argmax tokens match the golden trace within tolerance. This isolates the network from the sampler and from autoregressive drift.
6. Sampler + EOS alignment. Add the per-codebook sampling stack (repetition penalty, top-k/top-p/min-p, logit soft-cap) and free-running autoregression across all 9 codebooks. Evaluated by: seeded generation reproduces the golden token grid (or matches its distribution if seeds don't align across stacks), and eos_frame lands on the right frame.
7. DAC vocoder stage. Wire DAC decode → PCM @ 44.1 kHz. Evaluated by: feeding the golden token grid through your vocoder yields audio matching the reference PCM (spectral / sample-level distance under threshold). Confirms the codec seam independently of generation.
8. End-to-end quality parity. Connect all stages, full text→audio, no teacher forcing. Evaluated by: WER (via ASR) and speaker-similarity on a held-out set land within tolerance of the reference fork. This is the "works and sounds as good" gate.
9. Performance layer. Add CUDA-graph capture + torch.compile + batching covering the multi-codebook frame path, and chunked streaming PCM. Evaluated by: throughput and TTFB/RTF meet-or-beat the reference, while re-running step 8 shows no quality regression. Correctness and speed are checked separately so a speed change can't silently break audio.
10. Hardening + upstream deliverable. Lock in ragged-batch EOS edge cases, the full config surface (expose every knob, none hardcoded), and the model/topology contribution. Evaluated by: a CI suite that replays the per-stage golden checks from steps 3–7 as regression tests, all green — so any future change that drifts from the reference fails loudly.
